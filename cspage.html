Broke Bike Mountain Case Study

E-commerce Web application

Summary

Brokebike Mountain demonstrated a lot new techniques and tools used in creating a Fully Operational Web Application. The end result proved that these techniques could be learned and implemented professionally within the alotted time frame and still provide a quality site to be proud of. This Project was completed for my CareerFoundry course where it was consistantly built and tweaked over the duration of my studies

Tools Used

Heroku (Paas)

jQuery for star Rating system and product zoom

RSpec & FactoryBot for automated testing

SendGrid (to send order/sign up emails)

ActionCable for live comment and rating updates

Ruby for all application logic and database interaction

Ruby on Rails for web application deployment

Redis for caching

AJAX for user comments on merchandise

Brakeman for analyzing security vulnerabilities

Payment forms with the Stripe API

Devise gem for authentication of users

CanCanCan gem for authorization of users

Will_paginate gem for paginating comments

Goals

The main goal of this project was to use it as a platform to learn the concepts of a true Rails app and what it is capable of. Secondary to this was to create a production ready web application using Ruby on Rails. It would be able to sell products using stripe, accept and post user comments and distinguish between a registered user and a guest

Features

Show Page

View: User

The Show page gives a more detailed view of the selected product. The image can be zoomed for a more comprehensive inspection via the elementZoom plugin, you may also reach the purchasing option from here. If logged in you may leave a comment or rating which is also displayed as an average just under the image

View: Admin

The Admin page can be reached when logged in with an Administrator account. The differences here are the 'Edit & Destroy' buttons which give more control over the displayed content

Comments

View: User

The comments section shows a history of all posted comments and the associated rating given by that user. These comments are updated live with the use of ActionCable removing the need for a page refresh and giving the Web app a clean modern feel

View: Admin

With the Admin privilege comes the ability to delete unwanted comments. This can be done with a simple click on the black 'X' beside each comment. Once a user is signed in they may leave a comment whether as a User or an Admin

Product Page

View: User/Admin

The main product page shows all available products currently in the shop. From here a user may see the price, colour and description of each product. Selecting any will send the user to the associated Show page

Orders Page

View: Admin

The Orders page shows a list of all the current oders complete with order number, price, and the product description. When a user succesfully purchases a product via Stripe their order will be added to the list of standing orders

Contact Page

View: Admin

On the contact page is a simple, clean form enabling the user to contact the store. The messages are sent via mailer to a designated Email address along with the users information

Payment with Stripe

View: User

The Stripe API was used to allow safe and authorized payments with any major credit card
